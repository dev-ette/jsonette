Visibility Name Description
public parse(input: String): Result<JsonNode, Vec<Diagnostic>>

Strict parse. Either you get a valid tree or a list of errors. Used for validation, not live editing.
public tolerant_parse(input: String): (Option<JsonNode>, Vec<Diagnostic>)

Lenient parse. Gets called on every keystroke while the user is typing. Returns as much of the tree as it can parse, plus whatever errors it found. Critical: you don't want to throw away the whole tree because the user just deleted a }.
public format(node: JsonNode): String

Takes the tree, produces pretty-printed text. Shell calls this when the user hits the "Format" button, then replaces the editor's content.
public minify(node: JsonNode): String

Same but compact — strips all whitespace.
public evaluate_path(node: JsonNode, path: String): Result<Vec<JsonNode>, String>

Runs a JSONPath expression (e.g. $.users[*].email) against the tree. Returns all matching nodes or an error message if the path is invalid.
public completions_at(node: JsonNode, path_prefix: String): Vec<CompletionItem>

Given what the user has typed so far as a JSONPath prefix, walk the live tree and return what keys are valid at that position.
public diagnostics(input: String): Vec<Diagnostic>

Convenience shortcut: parse and return only errors, no tree. Used to refresh error markers without needing the full tree.
